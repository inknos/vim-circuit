1. Introduction                                   *claudeterm-introduction*

vim-claudeterm brings the Claude Code CLI directly into Vim using the
built-in |:terminal| feature. It manages a persistent terminal session
tied to your current project and git branch, and provides commands for
session switching, permission mode control, model selection, and more.

Key design principles:

  - Single persistent terminal per project/branch (no spawning duplicates)
  - Toggle show/hide without losing session state
  - Every command reachable three ways: long form, short alias, keymap
  - Every keymap and behavior configurable via |g:claudeterm_*| variables
  - Lifecycle hooks via |User| autocommands for extensibility

==============================================================================
2. Requirements                                   *claudeterm-requirements*

  - Vim 8.0+ compiled with |+terminal|
  - Claude Code CLI (`claude`) installed and in $PATH
  - `uuidgen` (for deterministic session IDs)
  - Git (for branch-based session strategy)

<

Native packages: >

    ~/.vim/pack/plugins/start/vim-claudeterm
<
After installing, run: >
  :helptags ALL
<

4. Quick Start                                     *claudeterm-quickstart*

Open a Claude terminal:                            *:CTerm* *:CT*
>
  :CTerm
  " or press <leader>c
<
The session persists -- toggling hides and restores the same terminal.

Resume an older session interactively:             *:CTerm-resume*
>
  :CTerm resume
  " or press <leader>cr
<
Switch to plan mode:                               *:CTerm-mode*
>
  :CTerm mode plan
  " or press <leader>cmp
<

5. Commands                                         *claudeterm-commands*

All commands support the dispatcher form `:CTerm {subcommand}` and short
aliases. Tab-completion is available for all subcommands and their
arguments.

------------------------------------------------------------------------------
5.1 Session Management                                *claudeterm-session*

                                                      *:CTerm-toggle*
:CTerm                  Toggle the Claude terminal. Opens a new session if
:CT                     none exists; shows/hides if one is already running.
                        Session ID is deterministic based on cwd and git
                        branch (see |g:claudeterm_session_strategy|).

                                                      *:CTresume*
:CTerm resume           Open the Claude interactive session picker
:CTresume               (`claude --resume`). Kills the current terminal
                        first if one is running.

                                                      *:CTcontinue*
:CTerm continue         Resume the most recent conversation in the current
:CTcontinue             directory (`claude --continue`).

                                                      *:CTnew*
:CTerm new              Kill the current session and start a brand new one.
:CTnew

                                                      *:CTkill*
:CTerm kill             Kill the Claude terminal buffer entirely.
:CTkill

                                                      *:CTpr*
:CTerm pr               Resume a session linked to a PR (`claude --from-pr`).
:CTpr                   Opens an interactive PR picker.

------------------------------------------------------------------------------
5.2 Mode Control                                       *claudeterm-modes*

Claude CLI has three interactive modes. These commands send the
corresponding slash command (`/plan`, `/fast`) to the running session --
no restart needed. If no session is running, one is opened first.

                                                      *:CTplan*
:CTerm plan             Toggle plan mode. Claude focuses on designing an
:CTplan                 implementation strategy without making changes. It
                        can research and explore the codebase but won't
                        edit files or write code. Toggle again (or send
                        `/plan`) to exit back to normal mode.

                                                      *:CTfast*
:CTerm fast             Toggle fast mode. Uses the same model but with
:CTfast                 faster output. Good for simpler tasks where speed
                        matters more than deep reasoning.

:CTerm mode {mode}      Send `/{mode}` to the running session. {mode} can
                        be `plan`, `fast`, or `normal` (which toggles plan
                        off if active).

------------------------------------------------------------------------------
5.3 Window Control                                    *claudeterm-window*

                                                      *:CTzoom*
:CTerm zoom             Toggle zoom: maximize the Claude terminal to
:CTzoom                 full-screen, or restore the previous split layout.
                        Works like tmux's `<prefix>z`.

                                                      *:CTerm-position*
:CTerm position {pos}   Move the terminal to a different split position.
                        {pos} is one of: `right`, `left`, `top`, `bottom`.
                        The terminal is hidden and re-shown at the new
                        position.

------------------------------------------------------------------------------
5.4 Code Context                                     *claudeterm-context*

                                                      *:CTsend*
:CTerm send             Send the current visual selection to the Claude
:CTsend                 terminal. The text is prefixed with the filename
                        for context. Use in visual mode.

                                                      *:CTchat*
:CTerm chat             Prompt for a free-form message and send it to
:CTchat                 Claude. The current alternate file name is included
                        as context. Opens the terminal if not visible.

------------------------------------------------------------------------------
5.5 Model & Config                                    *claudeterm-model*

                                                      *:CTmodel*
:CTerm model {name}     Kill the current session and restart with the given
:CTmodel {name}         model. {name} can be an alias (`sonnet`, `opus`,
                        `haiku`) or a full model name (e.g.
                        `claude-sonnet-4-6`).

                                                      *:CTverbose*
:CTerm verbose          Toggle verbose mode. Kills and restarts the session
:CTverbose              with or without `--verbose`.

------------------------------------------------------------------------------
5.6 Utility                                          *claudeterm-utility*

                                                      *:CTdoctor*
:CTerm doctor           Run `claude doctor` and display the health check
:CTdoctor               results in the message area.

                                                      *:CTversion*
:CTerm version          Show the plugin version, Claude CLI version, Vim
:CTversion              version, and +terminal support status.

6. Keymaps                                           *claudeterm-keymaps*

All keymaps are registered only if |g:claudeterm_map_keys| is 1 (default).
Every keymap is configurable via the corresponding `g:claudeterm_map_*`
variable.

------------------------------------------------------------------------------
6.1 Normal Mode                                *claudeterm-keymaps-normal*

  Key              Command         Variable ~
  <leader>c        Toggle          |g:claudeterm_map_toggle|
  <leader>cr       Resume          |g:claudeterm_map_resume|
  <leader>cc       Continue        |g:claudeterm_map_continue|
  <leader>cn       New session     |g:claudeterm_map_new|
  <leader>ck       Kill            |g:claudeterm_map_kill|
  <leader>cp       From PR         |g:claudeterm_map_pr|
  <leader>cz       Zoom toggle     |g:claudeterm_map_zoom|
  <leader>ch       Chat            |g:claudeterm_map_chat|
  <leader>cv       Verbose toggle  |g:claudeterm_map_verbose|
  <leader>cmp      Mode: plan      |g:claudeterm_map_mode_plan|
  <leader>cmf      Mode: fast      |g:claudeterm_map_mode_fast|
  <leader>cms      Model: sonnet   |g:claudeterm_map_model_sonnet|
  <leader>cmo      Model: opus     |g:claudeterm_map_model_opus|
  <leader>cmh      Model: haiku    |g:claudeterm_map_model_haiku|

------------------------------------------------------------------------------
6.2 Visual Mode                                *claudeterm-keymaps-visual*

  Key              Command         Variable ~
  <leader>cs       Send selection  |g:claudeterm_map_send|

------------------------------------------------------------------------------
6.3 Terminal Mode                            *claudeterm-keymaps-terminal*

These keymaps are active when the cursor is inside any terminal buffer.

  Key              Action ~
  <C-w>z           Zoom toggle (|g:claudeterm_map_zoom_term|)
  <C-h>            Navigate to window left
  <C-j>            Navigate to window below
  <C-k>            Navigate to window above
  <C-l>            Navigate to window right
  <C-v>            Paste from system clipboard

==============================================================================
7. Tab-Completion                                  *claudeterm-completion*

`:CTerm` supports context-aware tab-completion:

  :CTerm <Tab>           Completes subcommands: `resume`, `continue`,
                         `new`, `kill`, `pr`, `plan`, `fast`, `mode`,
                         `zoom`, `position`, `send`, `chat`, `model`,
                         `verbose`, `doctor`, `version`.

  :CTerm mode <Tab>      Completes: `plan`, `fast`, `normal`.

  :CTerm model <Tab>     Completes: `sonnet`, `opus`, `haiku`.

  :CTerm position <Tab>  Completes: `right`, `left`, `top`, `bottom`.

8. Hooks                                               *claudeterm-hooks*

The plugin fires |User| autocommands with a `ClaudeTerm` prefix at various
lifecycle points. Hook into them in your |vimrc|:
>
  autocmd User ClaudeTermOpen echo "Claude session started"
  autocmd User ClaudeTermReload echohl WarningMsg
        \ | echo "Buffers reloaded" | echohl None
<

Available events:

  Event                   When fired ~
  ClaudeTermOpen          Terminal buffer created for the first time
  ClaudeTermToggleShow    Terminal shown (unhidden)
  ClaudeTermToggleHide    Terminal hidden
  ClaudeTermKill          Terminal killed
  ClaudeTermZoomIn        Terminal zoomed to full-screen
  ClaudeTermZoomOut       Terminal restored from zoom
  ClaudeTermReload        One or more buffers auto-reloaded
  ClaudeTermModeChange    Permission mode changed
  ClaudeTermSessionChange Session switched (resume/continue/new/pr)

9. Configuration                               *claudeterm-configuration*

All variables use the `g:claudeterm_` prefix. Buffer-local `b:claudeterm_*`
overrides take precedence over global values.

Set variables in your |vimrc| before the plugin loads, or at any time
(most take effect on the next command invocation).

------------------------------------------------------------------------------
9.1 Core                                        *claudeterm-config-core*

                                                *g:claudeterm_command*
g:claudeterm_command          String (default: `'claude'`)
                              CLI executable name or full path.

                                                *g:claudeterm_position*
g:claudeterm_position         String (default: `'right'`)
                              Window position: `'right'`, `'left'`, `'top'`,
                              `'bottom'`.

                                                *g:claudeterm_split_ratio*
g:claudeterm_split_ratio      Float (default: `0.4`)
                              Terminal size as a fraction of the screen
                              (0.0 to 1.0).

                                                *g:claudeterm_enter_insert*
g:claudeterm_enter_insert     Number (default: `1`)
                              Auto-enter terminal (insert) mode when the
                              Claude window gains focus.

                                                *g:claudeterm_use_git_root*
g:claudeterm_use_git_root     Number (default: `1`)
                              Start Claude at the git repository root
                              instead of the current working directory.

                                                *g:claudeterm_permission_mode*
g:claudeterm_permission_mode  String (default: `''`)
                              Default `--permission-mode` passed at CLI
                              startup. Controls tool approval behavior.
                              Empty means CLI default. Valid values:
                              `'default'`, `'plan'`, `'auto'`,
                              `'acceptEdits'`, `'dontAsk'`,
                              `'bypassPermissions'`.
                              Note: this is separate from the interactive
                              modes toggled by |:CTplan| and |:CTfast|.

                                                *g:claudeterm_model*
g:claudeterm_model            String (default: `''`)
                              Default `--model` override. Empty means CLI
                              default. Can be an alias (`'sonnet'`) or a
                              full name (`'claude-sonnet-4-6'`).

                                                *g:claudeterm_extra_args*
g:claudeterm_extra_args       String (default: `''`)
                              Extra CLI arguments appended to every `claude`
                              invocation. Example: `'--add-dir /tmp/data'`.

------------------------------------------------------------------------------
9.2 Session                                   *claudeterm-config-session*

                                                *g:claudeterm_session_strategy*
g:claudeterm_session_strategy String (default: `'branch'`)
                              How the deterministic session ID is computed:
                              `'branch'`  - hash of cwd + git branch
                              `'directory'` - hash of cwd only
                              `'none'`    - let the CLI decide (no
                                            `--session-id` flag)

                                                *g:claudeterm_open_strategy*
g:claudeterm_open_strategy    String (default: `'resume'`)
                              What happens when you toggle open (|:CTerm|)
                              and no terminal is running:

                              `'resume'` - Resume the most recent
                                conversation in the current directory
                                (`claude --continue`). If no prior session
                                exists, Claude starts a new conversation
                                automatically.

                              `'fresh'`  - Always start a new session with
                                a deterministic session ID based on the
                                |g:claudeterm_session_strategy|. Resumes
                                that specific session if it already exists.

                              This only affects the toggle command. Explicit
                              commands like |:CTresume|, |:CTcontinue|, and
                              |:CTnew| always do what they say regardless
                              of this setting.

                              Example: >
                                let g:claudeterm_open_strategy = 'fresh'
<

------------------------------------------------------------------------------
9.3 Auto-Reload                                *claudeterm-config-reload*

                                                *g:claudeterm_auto_reload*
g:claudeterm_auto_reload      Number (default: `1`)
                              Enable buffer auto-reload polling while the
                              Claude terminal is active. Uses a timer to
                              detect files changed on disk and runs
                              |:checktime| on them.

                                                *g:claudeterm_reload_interval*
g:claudeterm_reload_interval  Number (default: `1000`)
                              Polling interval in milliseconds.

                                                *g:claudeterm_notify_reload*
g:claudeterm_notify_reload    Number (default: `1`)
                              Echo a notification when buffers are reloaded.

------------------------------------------------------------------------------
9.4 Keymaps                                   *claudeterm-config-keymaps*

                                                *g:claudeterm_map_keys*
g:claudeterm_map_keys         Number (default: `1`)
                              Set to `0` to disable all default keymaps.
                              You can then define your own.

Each keymap has its own variable. See |claudeterm-keymaps| for the full
list. Example override:
>
  let g:claudeterm_map_toggle = '<F5>'
  let g:claudeterm_map_zoom = '<leader>z'
<

------------------------------------------------------------------------------
9.5 Display                                   *claudeterm-config-display*

                                                *g:claudeterm_hide_numbers*
g:claudeterm_hide_numbers     Number (default: `1`)
                              Hide line numbers in the terminal window.

                                                *g:claudeterm_hide_signcolumn*
g:claudeterm_hide_signcolumn  Number (default: `1`)
                              Hide the sign column in the terminal window.

10. Troubleshooting                           *claudeterm-troubleshooting*

Terminal does not open~
  Verify `vim --version | grep +terminal`. This plugin requires Vim
  compiled with +terminal.

Claude not found~
  Ensure `claude` is in `$PATH`. Run `:CTdoctor` for a health check.

Session ID errors~
  The plugin uses `uuidgen --sha1`. Ensure `uuidgen` is installed.
  On most Linux distributions it is part of `util-linux`.

Buffers not reloading~
  Check that `g:claudeterm_auto_reload` is `1` and that |'autoread'| is
  not globally disabled. You can also set `set autoread` in your vimrc.

Keymaps conflict~
  Set `g:claudeterm_map_keys = 0` to disable all defaults, then define
  your own mappings. Or override individual keys via the
  `g:claudeterm_map_*` variables.
