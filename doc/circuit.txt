1. Introduction                                   *circuit-introduction*

vim-circuit brings any agent CLI directly into Vim using the built-in
|:terminal| feature. It manages a persistent terminal session tied to
your current project and git branch, and provides commands for session
switching, permission mode control, model selection, and more.

Key design principles:

  - Single persistent terminal per project/branch (no spawning duplicates)
  - Toggle show/hide without losing session state
  - Multi-provider: works with Claude, Cursor Agent, Gemini, OpenCode
  - Every command reachable three ways: long form, short alias, keymap
  - Every keymap and behavior configurable via |g:circuit_*| variables
  - Lifecycle hooks via |User| autocommands for extensibility

==============================================================================
2. Providers                                       *circuit-providers*

vim-circuit supports multiple agent CLI backends. Set
|g:circuit_provider| in your |vimrc| to choose which one to use.

                                                *g:circuit_provider*
g:circuit_provider      String (default: `'claude'`)
                        The active provider. Valid values:
                        `'claude'`, `'agent'`, `'gemini'`, `'opencode'`.

Available providers:

  Provider          Binary      Project ~
  Claude Code       `claude`    Anthropic Claude Code CLI
  Cursor Agent      `agent`     Cursor Agent CLI
  Gemini CLI        `gemini`    Google Gemini CLI
  OpenCode          `opencode`  OpenCode (recommended)

vim-circuit does not handle authentication or API keys. Each CLI must
already be installed, authenticated, and working on its own before you
use it through this plugin.

Feature matrix:                            *circuit-provider-features*

  Feature                 Claude  Agent  Gemini  OpenCode ~
  Continue session          ✓       ✓      ✓       ✓
  Resume (picker)           ✓       ✓      ✓       ✓
  Model switching           ✓       ✓      ✓       ✓
  Permission/approval       ✓       ✓      ✓       —
  Verbose/debug             ✓       —      ✓       —
  Worktree                  ✓       ✓      ✓       —
  From PR                   ✓       —      —       —
  Interactive modes         ✓       ✓      —       —
  Health check              ✓       ✓      —       —
  Undo/redo                 —       —      —       ✓
  Export                    —       —      —       ✓
  Stats                     —       —      —       ✓
  Session list              —       ✓      ✓       ✓

When you use a feature not supported by your provider, the plugin
prints a warning instead of failing silently.

==============================================================================
3. Requirements                                   *circuit-requirements*

  - Vim 8.0+ compiled with |+terminal|
  - A supported agent CLI installed and in $PATH (see |circuit-providers|)
  - The CLI must be authenticated and functional on its own
  - Git

4. Installation                                   *circuit-installation*

vim-plug: >
  Plug 'inknos/vim-circuit'
<
Native packages: >
  git clone https://github.com/inknos/vim-circuit.git \
    ~/.vim/pack/plugins/start/vim-circuit
<
After installing, run: >
  :helptags ALL
<

5. Quick Start                                     *circuit-quickstart*

Set your provider in your vimrc:                  *:CTerm* *:CT*
>
  let g:circuit_provider = 'opencode'
<
Open an agent terminal:
>
  :CTerm
  " or press <leader>c
<
The session persists -- toggling hides and restores the same terminal.

Resume an older session interactively:             *:CTerm-resume*
>
  :CTerm resume
  " or press <leader>cr
<
Switch model:                                      *:CTerm-model*
>
  :CTerm model {name}
<
Send interactive mode command (provider-dependent): *:CTerm-mode*
>
  :CTerm mode plan
  " or press <leader>cmp
<

6. Commands                                         *circuit-commands*

All commands support the dispatcher form `:CTerm {subcommand}` and short
aliases. Tab-completion is available for all subcommands and their
arguments. Features unsupported by the active provider print a warning.

------------------------------------------------------------------------------
6.1 Session Management                                *circuit-session*

                                                      *:CTerm-toggle*
:CTerm                  Toggle the agent terminal. Resumes the most recent
:CT                     session (`--continue` or equivalent), or starts a
                        new one if none exists. Shows/hides if a session
                        is already running.

                                                      *:CTresume*
:CTerm resume           Open the interactive session picker. Kills the
:CTresume               current terminal first if one is running.

                                                      *:CTcontinue*
:CTerm continue         Resume the most recent conversation in the current
:CTcontinue             directory.

                                                      *:CTnew*
:CTerm new              Kill the current session and start a brand new one.
:CTnew

                                                      *:CTkill*
:CTerm kill             Kill the terminal buffer entirely.
:CTkill

                                                      *:CTpr*
:CTerm pr               Resume a session linked to a PR. Claude only;
:CTpr                   other providers print a warning.

------------------------------------------------------------------------------
6.2 Mode Control                                       *circuit-modes*

The agent CLI may support interactive modes. These commands send the
corresponding slash command to the running session -- no restart needed.
If no session is running, one is opened first. Providers without mode
support print a warning.

                                                      *:CTplan*
:CTerm plan             Toggle plan mode (sends `/{mode_prefix}plan`).
:CTplan

                                                      *:CTfast*
:CTerm fast             Toggle fast mode (sends `/{mode_prefix}fast`).
:CTfast

:CTerm mode {mode}      Send the provider's mode prefix + {mode} to the
                        running session. Available modes depend on the
                        active provider (tab-completable).

------------------------------------------------------------------------------
6.3 Window Control                                    *circuit-window*

                                                      *:CTzoom*
:CTerm zoom             Toggle zoom: maximize the terminal to full-screen,
:CTzoom                 or restore the previous split layout.

                                                      *:CTerm-position*
:CTerm position {pos}   Move the terminal to a different split position.
                        {pos} is one of: `right`, `left`, `top`, `bottom`.

------------------------------------------------------------------------------
6.4 Code Context                                     *circuit-context*

                                                      *:CTsend*
:CTerm send             Send the current visual selection to the agent
:CTsend                 terminal. The text is prefixed with the filename.

                                                      *:CTchat*
:CTerm chat             Prompt for a free-form message and send it to the
:CTchat                 agent. Opens the terminal if not visible.

------------------------------------------------------------------------------
6.5 Worktree                                         *circuit-worktree*

                                                      *:CTworktree*
:CTworktree [name]      Launch the agent in a git worktree. Providers
:CTerm worktree [name]  without worktree support print a warning.

:CTworktree! [name]     With a bang, adds the tmux flag (if the provider
                        supports it). Also triggered when
                        |g:circuit_worktree_tmux| is `1`.

------------------------------------------------------------------------------
6.6 Model & Config                                    *circuit-model*

                                                      *:CTmodel*
:CTerm model {name}     Kill the current session and restart with the given
:CTmodel {name}         model. Available model aliases depend on the active
                        provider (tab-completable). Full model names are
                        always accepted.

                                                      *:CTverbose*
:CTerm verbose          Toggle verbose/debug mode. Providers without
:CTverbose              verbose support print a warning.

------------------------------------------------------------------------------
6.7 Undo / Redo                                        *circuit-undo*

                                                      *:CTundo*
:CTerm undo             Send the provider's undo slash command to the
:CTundo                 running terminal. Currently only OpenCode supports
                        this feature. Other providers print a warning.

                                                      *:CTredo*
:CTerm redo             Send the provider's redo slash command to the
:CTredo                 running terminal.

------------------------------------------------------------------------------
6.8 Export                                            *circuit-export*

                                                      *:CTexport*
:CTerm export           Send the provider's export slash command to the
:CTexport               running terminal. Currently only OpenCode supports
                        this feature.

------------------------------------------------------------------------------
6.9 Stats                                              *circuit-stats*

                                                      *:CTstats*
:CTerm stats            Run the provider's stats CLI subcommand and display
:CTstats                the output. Currently only OpenCode supports this
                        feature.

------------------------------------------------------------------------------
6.10 Session List                                   *circuit-sessions*

                                                      *:CTsessions*
:CTerm sessions         Open the provider's session list interface. For
:CTsessions             providers without a session list command, falls
                        back to |:CTresume|.

7. Keymaps                                           *circuit-keymaps*

All keymaps are registered only if |g:circuit_map_keys| is 1 (default).
Every keymap is configurable via the corresponding `g:circuit_map_*`
variable.

------------------------------------------------------------------------------
7.1 Normal Mode                                *circuit-keymaps-normal*

  Key              Command         Variable ~
  <leader>c        Toggle          |g:circuit_map_toggle|
  <leader>cr       Resume          |g:circuit_map_resume|
  <leader>cc       Continue        |g:circuit_map_continue|
  <leader>cn       New session     |g:circuit_map_new|
  <leader>ck       Kill            |g:circuit_map_kill|
  <leader>cp       From PR         |g:circuit_map_pr|
  <leader>cz       Zoom toggle     |g:circuit_map_zoom|
  <leader>ch       Chat            |g:circuit_map_chat|
  <leader>cv       Verbose toggle  |g:circuit_map_verbose|
  <leader>cw       Worktree        |g:circuit_map_worktree|
  <leader>cmp      Mode: plan      |g:circuit_map_mode_plan|
  <leader>cmf      Mode: fast      |g:circuit_map_mode_fast|
  <leader>cu       Undo            |g:circuit_map_undo|
  <leader>cU       Redo            |g:circuit_map_redo|
  <leader>ce       Export          |g:circuit_map_export|
  <leader>cl       Session list    |g:circuit_map_sessions|

------------------------------------------------------------------------------
7.2 Visual Mode                                *circuit-keymaps-visual*

  Key              Command         Variable ~
  <leader>cs       Send selection  |g:circuit_map_send|

------------------------------------------------------------------------------
7.3 Terminal Mode                            *circuit-keymaps-terminal*

These keymaps are active when the cursor is inside any terminal buffer.

  Key              Action ~
  <C-w>z           Zoom toggle (|g:circuit_map_zoom_term|)
  <C-h>            Navigate to window left
  <C-j>            Navigate to window below
  <C-k>            Navigate to window above
  <C-l>            Navigate to window right
  <C-v>            Paste from system clipboard

==============================================================================
8. Tab-Completion                                  *circuit-completion*

`:CTerm` supports context-aware tab-completion:

  :CTerm <Tab>           Completes subcommands: `resume`, `continue`,
                         `new`, `kill`, `pr`, `worktree`, `plan`,
                         `fast`, `mode`, `zoom`, `position`, `send`,
                         `chat`, `model`, `verbose`, `doctor`,
                         `version`, `undo`, `redo`, `export`,
                         `stats`, `sessions`.

  :CTerm mode <Tab>      Completes from the active provider's mode list.
                         E.g. `plan`, `fast`, `normal` for Claude;
                         `plan`, `ask` for Cursor Agent; empty for
                         OpenCode and Gemini.

  :CTerm model <Tab>     Completes from the active provider's model list.
                         E.g. `sonnet`, `opus`, `haiku` for Claude;
                         `pro`, `flash`, `flash-lite` for Gemini;
                         empty for OpenCode.

  :CTerm position <Tab>  Completes: `right`, `left`, `top`, `bottom`.

9. Hooks                                               *circuit-hooks*

The plugin fires |User| autocommands with a `CT` prefix at various
lifecycle points. Hook into them in your |vimrc|:
>
  autocmd User CTOpen echo "Agent session started"
  autocmd User CTReload echohl WarningMsg
        \ | echo "Buffers reloaded" | echohl None
<

Available events:

  Event                   When fired ~
  CTOpen                  Terminal buffer created for the first time
  CTToggleShow            Terminal shown (unhidden)
  CTToggleHide            Terminal hidden
  CTKill                  Terminal killed
  CTTermExited            Agent process exited (split closed, buffer wiped)
  CTZoomIn                Terminal zoomed to full-screen
  CTZoomOut               Terminal restored from zoom
  CTReload                One or more buffers auto-reloaded
  CTModeChange            Interactive mode changed
  CTSessionChange         Session switched (resume/continue/new/pr)
  CTWorktree              Worktree session launched
  CTUndo                  Undo slash command sent
  CTRedo                  Redo slash command sent
  CTExport                Export slash command sent
  CTSessionList           Session list opened

10. Configuration                              *circuit-configuration*

All variables use the `g:circuit_` prefix. Buffer-local `b:circuit_*`
overrides take precedence over global values.

Set variables in your |vimrc| before the plugin loads, or at any time
(most take effect on the next command invocation).

------------------------------------------------------------------------------
10.1 Core                                       *circuit-config-core*

                                                *g:circuit_provider*
g:circuit_provider          String (default: `'claude'`)
                            The active provider backend. See
                            |circuit-providers| for valid values and the
                            feature matrix. OpenCode is recommended.

                                                *g:circuit_command*
g:circuit_command           String (default: `''`)
                            Override the CLI binary name or path. When
                            empty (default), the provider's default binary
                            is used (e.g. `claude` for the claude provider,
                            `opencode` for the opencode provider).

                                                *g:circuit_position*
g:circuit_position          String (default: `'right'`)
                            Window position: `'right'`, `'left'`, `'top'`,
                            `'bottom'`.

                                                *g:circuit_split_ratio*
g:circuit_split_ratio       Float (default: `0.4`)
                            Terminal size as a fraction of the screen.

                                                *g:circuit_enter_insert*
g:circuit_enter_insert      Number (default: `1`)
                            Auto-enter terminal (insert) mode when the
                            agent window gains focus.

                                                *g:circuit_use_git_root*
g:circuit_use_git_root      Number (default: `1`)
                            Start the agent at the git repository root.

                                                *g:circuit_permission_mode*
g:circuit_permission_mode   String (default: `''`)
                            Default permission/approval mode passed at CLI
                            startup. The actual flag used depends on the
                            provider (e.g. `--permission-mode` for Claude,
                            `--mode` for Cursor Agent, `--approval-mode`
                            for Gemini). Empty means CLI default.

                                                *g:circuit_model*
g:circuit_model             String (default: `''`)
                            Default model override. Empty means CLI
                            default. Can be a provider-specific alias or
                            a full model name.

                                                *g:circuit_extra_args*
g:circuit_extra_args        String (default: `''`)
                            Extra CLI arguments appended to every agent
                            invocation.

                                                *g:circuit_worktree_tmux*
g:circuit_worktree_tmux     Number (default: `0`)
                            When `1`, `:CTworktree` always passes the
                            provider's tmux flag. Can also be toggled
                            per-call with `:CTworktree!`.

------------------------------------------------------------------------------
10.2 Auto-Reload                               *circuit-config-reload*

                                                *g:circuit_auto_reload*
g:circuit_auto_reload       Number (default: `1`)
                            Enable buffer mtime auto-reload while the agent
                            terminal is active. The same timer (see
                            |g:circuit_reload_interval|) still runs when
                            this is `0` so the plugin can detect a finished
                            job and close the terminal split (see
                            |CTTermExited|). |'autoread'| and
                            |g:circuit_notify_reload| only apply when this
                            is `1`.

                                                *g:circuit_reload_interval*
g:circuit_reload_interval   Number (default: `1000`)
                            Polling interval in milliseconds for the
                            timer that checks for a finished job and, when
                            |g:circuit_auto_reload| is on, for buffer
                            mtime changes.

                                                *g:circuit_notify_reload*
g:circuit_notify_reload     Number (default: `1`)
                            Echo a notification when buffers are reloaded.

------------------------------------------------------------------------------
10.3 Keymaps                                  *circuit-config-keymaps*

                                                *g:circuit_map_keys*
g:circuit_map_keys          Number (default: `1`)
                            Set to `0` to disable all default keymaps.

Each keymap has its own variable. See |circuit-keymaps| for the full
list. Example override:
>
  let g:circuit_map_toggle = '<F5>'
  let g:circuit_map_zoom = '<leader>z'
<

------------------------------------------------------------------------------
10.4 Display                                  *circuit-config-display*

                                                *g:circuit_hide_numbers*
g:circuit_hide_numbers      Number (default: `1`)
                            Hide line numbers in the terminal window.

                                                *g:circuit_hide_signcolumn*
g:circuit_hide_signcolumn   Number (default: `1`)
                            Hide the sign column in the terminal window.

11. Troubleshooting                          *circuit-troubleshooting*

Terminal does not open~
  Verify `vim --version | grep +terminal`. This plugin requires Vim
  compiled with +terminal.

Agent CLI not found~
  Ensure the agent CLI is in `$PATH`. Check that |g:circuit_provider|
  matches the CLI you have installed. Run `:CTdoctor` for a health
  check (if supported by your provider).

Unsupported feature warning~
  Not all features work with all providers. See |circuit-providers|
  for the feature matrix. The plugin warns instead of erroring when
  you use an unsupported feature.

Buffers not reloading~
  Check that `g:circuit_auto_reload` is `1` and that |'autoread'| is
  not globally disabled.

Empty window after the agent exits~
  When the agent CLI (e.g. opencode) exits, the split should close
  automatically (|TermClose| or the circuit timer). If a stale buffer
  remains, run |:CTkill| or `:bwipe` that buffer.

OpenCode prints JSON (`sessionID`, `invalid_format`, `ses`, etc.)~
  This comes from the `opencode` process, not from vim-circuit. The
  plugin only runs the shell command (e.g. `opencode --continue` for
  |:CT|). Some OpenCode versions or features (e.g. subagent dispatch)
  validate session IDs and may log a JSON error if a placeholder such
  as `dummy` is used where a real ID (starting with `ses`) is
  required. Workarounds: upgrade `opencode` to the latest release, try
  |:CTnew| to start without `--continue`, and check OpenCode
  project issues for the exact message. If the session still works,
  the line is often harmless noise from a background request.

Keymaps conflict~
  Set `g:circuit_map_keys = 0` to disable all defaults, then define
  your own mappings.
